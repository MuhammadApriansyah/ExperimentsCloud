# Architecture Decision Records (ADR)

This directory contains the Architecture Decision Records (ADR) for ExperimentsCloud.

## Purpose

Each ADR documents an important architectural decision, including:

- Context
- Decision
- Decision Drivers
- Consequences
- Alternatives Considered

## Numbering

ADR identifiers are permanent.

- Numbers are never reused.
- Deleted ADRs are not renumbered.
- New ADRs always use the next available number.

## Status

Each ADR must include one of the following statuses:

- Proposed
- Accepted
- Superseded
- Deprecated

## Template

Each ADR should follow this structure:

- Title
- Status
- Date
- Authors
- Context
- Decision Drivers
- Decision
- Consequences
- Alternatives Considered
- Future Work (optional)
